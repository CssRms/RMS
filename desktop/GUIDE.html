ZKTeco Attendance Monitor

A desktop application that captures fingerprint attendance from a ZKTeco biometric device over your local network, stores records locally, and syncs them to your cloud portal on Railway.

DeviceZKTeco MB160 (ADMS)
PlatformWindows 10 / 11
RequiresPython 3.10+
Cloudcssgrouprms.com

How it works

The flow has three stages: the device captures the fingerprint → the app on your laptop receives it → you sync to the cloud when ready.

ZKTeco Device          Your Laptop              Railway Cloud
(fingerprint scan)     (this app)               (cssgrouprms.com)
      │                     │                          │
      │── scan detected ──► │                          │
      │   sends Staff ID    │── saves to SQLite ──►    │
      │   + timestamp       │   dashboard shows live   │
      │                     │                          │
      │                     │── "Upload to Railway" ──►│
      │                     │   syncs all pending      │
      │                     │   records at once        │

The Staff ID is the key that links everything. The same ID must be set on the device (User ID when enrolling a fingerprint), in the desktop app's employee list, and in the Railway portal's Employee Directory.

STEP 01Installation

Install Python

Python is the programming language the app runs on. You only do this once.

  1. Open your browser and go to https://www.python.org/downloads/
  2. Click the big yellow Download Python button — it picks the right version for you.
  3. Run the installer. On the first screen, tick "Add Python to PATH" before clicking Install Now.

    Important
    If you miss the "Add Python to PATH" checkbox, the app won't start. Run the installer again and tick it.
  4. When it finishes, open Command Prompt (press Win + R, type cmd, press Enter) and type: 
    python --version
    You should see something like Python 3.13.0. If you see an error, Python wasn't added to PATH — reinstall and tick the box.
STEP 02Installation

Get the app files

The desktop app lives in the desktop/ folder inside the main RMS project on GitHub.

Option A — Download the folder (easiest)

  1. Go to github.com/CssRms/RMS
  2. Click the green Code button → Download ZIP
  3. Unzip it anywhere, e.g. C:\RMS\
  4. The app files are in the desktop\ sub-folder inside the unzipped folder.

Option B — Git clone (recommended for updates)

# Open Command Prompt, then:
git clone https://github.com/CssRms/RMS.git C:\RMS

Your app folder will be at C:\RMS\desktop\.

Tip — Where to put it
Put the folder somewhere permanent like C:\RMS\desktop\. If you later build the .exe (Step 12) you can put that exe anywhere, but the source files should stay in one place.
STEP 03Installation

Install dependencies

The app uses several Python libraries. Install them all with one command.

  1. Open Command Prompt as Administrator (right-click Command Prompt → Run as administrator).
  2. Navigate to the desktop folder: 
    cd C:\RMS\desktop
  3. Install all required libraries: 
    pip install -r requirements.txt
    This downloads and installs Flask, SQLAlchemy, openpyxl, pystray, Pillow, and everything else. It may take 1–2 minutes.
  4. When it finishes without errors, you're ready.
If pip is not found
Try python -m pip install -r requirements.txt instead. If that also fails, Python wasn't added to PATH — go back to Step 1.
STEP 04Device Setup

Configure the ZKTeco device

The device needs to know the address of your laptop so it can send attendance records to it.

Before you start — find your laptop's IP address
Open Command Prompt and type ipconfig. Look for IPv4 Address under your Wi-Fi adapter. It will look like 192.168.1.50. Write it down — you'll need it below.

On the ZKTeco device

  1. Press Menu on the device (or use the touch screen menu button).
  2. Go to Communication → Cloud Server Settings (or ADMS depending on firmware).
  3. Set the following values:
    SettingValue to enter
    Server Mode ADMS REQUIRED
    Enable Domain Name / VDomain OFF (disabled) REQUIRED
    Server Address 192.168.1.50
    (your laptop's IP from ipconfig)    		 REQUIRED
    Server Port 80 REQUIRED
    Enable Proxy Server OFF (disabled) REQUIRED
  4. Save and exit the settings.
  5. Reboot the device (Menu → System → Reboot, or unplug and plug back in).
Important — Wi-Fi network
Both your laptop AND the ZKTeco device must be connected to the same Wi-Fi network. If the device is on one router and the laptop on another, they cannot communicate.
When to use cssgrouprms.com instead
The settings above (laptop IP) are for local mode — capturing on your laptop first, then syncing to the cloud later. If you want the device to push directly to Railway cloud, set Server Address to cssgrouprms.com, enable Domain Name (VDomain ON), and leave Port hidden. See Step 11 for details.
STEP 05Device Setup

Register employees in the app

Every employee must be registered in the app with their Staff ID — the same ID they were given on the device.

Using the dashboard (recommended)

Once the app is running (Step 7), open the dashboard at http://localhost and use the Register Employee form in the left sidebar:

FieldWhat to enter
Staff IDThe User ID you set on the device when enrolling (e.g. 11112222). This must match exactly.
Full NameThe employee's full name (e.g. Ephraim Hosea).
DepartmentOptional — e.g. ICT, HR, Admin.
PositionOptional — e.g. Officer, Manager.

Click Save. The employee appears in the left list immediately.

Using the seed script (bulk load)

If you have many employees, edit the list in seed.py and run it once:

# Edit desktop/seed.py first — add all your staff
# Then run:
python seed.py
The Golden Rule
The Staff ID in the app must exactly match the User ID on the device. This is the only way the system knows whose fingerprint was scanned.
STEP 06Device Setup

Enroll fingerprints on the device

Fingerprint enrollment is physical — it must be done directly on the ZKTeco device. No software can do this remotely.

  1. On the device, go to Menu → User Management → New User (or Add User).
  2. Set the User ID to the Staff ID you registered in Step 5 (e.g. 11112222).
  3. Set the Name to the employee's name.
  4. Select Fingerprint and follow the on-screen prompts — the employee presses their finger on the sensor 3 times.
  5. Save. The device now knows that fingerprint belongs to User ID 11112222.
Can an employee use a PIN instead?
Yes. Set a PIN on the device for that User ID. When they enter their PIN, the device sends the same User ID — the app handles it identically to a fingerprint scan.
STEP 07Daily Use

Start the app

You only need to do this once per day (or set it to auto-start — see Step 13).

While running as a Python script (before building .exe)

  1. Open Command Prompt as Administrator.
  2. Navigate to the desktop folder: 
    cd C:\RMS\desktop
  3. Run the app: 
    python app.py
  4. A green fingerprint icon 👁 appears in the system tray (bottom-right corner, near the clock).
  5. Your browser opens automatically to the dashboard at http://localhost.

After building the .exe (Step 12)

Simply double-click ZKAttendance.exe. The tray icon appears and the browser opens — no Command Prompt needed.

Port 80 needs Administrator
The server runs on port 80 which is a privileged port on Windows. Always run Command Prompt as Administrator, or use "Run as Administrator" when double-clicking the .exe. If you get a "Permission denied" error, this is why.
STEP 08Daily Use

Run diagnostics before attendance starts

Always run diagnostics at the start of each day to confirm everything is working before staff begin clocking in.

  1. With the app running, open the dashboard in your browser (http://localhost).
  2. Click the Diagnostics tab at the top.
  3. Click ▶ Run Diagnostics.
  4. Wait a few seconds. You will see a result for each check:
    CheckWhat it verifies
    ZKTeco DeviceIs the device sending heartbeats to this laptop? (means the IP/port is correct)
    Laptop NetworkShows your LAN IP — confirm it matches what's set on the device in Step 4
    Employee DatabaseHow many employees are registered, scans today, records pending sync
    Railway CloudCan this laptop reach cssgrouprms.com for syncing?
    Excel ExportIs the Excel library installed for report downloads?
  5. If the overall result is ✅ All Systems Ready, you can start attendance.
If "ZKTeco Device" shows ❌ or ⚠️
The device hasn't connected yet. Check: (1) device is powered on and connected to Wi-Fi, (2) the Server Address on the device matches your laptop's LAN IP from ipconfig, (3) port is set to 80, (4) proxy is OFF. Then reboot the device and wait 30 seconds, then run diagnostics again.
STEP 09Daily Use

Taking attendance

Once diagnostics pass, attendance runs automatically — staff just scan their fingerprints on the device.

What you see

On the Live Attendance tab, every scan triggers an instant update:

  • The Last Scan card flashes with the employee's name, department, time, and status (Check In / Check Out).
  • The employee's avatar in the left sidebar turns green with the scan time.
  • A new row is added to Today's Log at the bottom.
  • The Scans Today counter on the right increments.

Unknown fingerprint scanned

If a fingerprint is registered on the device but not in the app's employee list, you'll see an ⚠ Unknown Fingerprint card with the device User ID. Go to the Register Employee form and add that employee — future scans by that person will be recognised.

Keep the dashboard open on a screen
Many offices keep the dashboard open on a monitor or TV near the entrance so they can see attendance happening live.
STEP 10Daily Use

Export attendance to Excel

Download a formatted Excel spreadsheet of any day's attendance.

  1. In the dashboard's right sidebar, find the Export to Excel section.
  2. The date field defaults to today. Change it if you want a different day.
  3. Click ⬇ XLS. The file downloads automatically.
  4. Open it in Microsoft Excel or Google Sheets.

The file contains: Staff ID, Name, Department, Position, Time, Status (Check In/Out), and whether the record has been synced to Railway Cloud.

No openpyxl installed?
The export falls back to a .csv file which also opens in Excel. To get proper .xlsx formatting, run pip install openpyxl in Command Prompt.
STEP 11Daily Use

Sync attendance to Railway Cloud

When you're ready to send today's (or any pending) records to the cloud portal, press the sync button.

  1. Make sure you have an internet connection.
  2. In the dashboard's right sidebar, click ↑ Upload to Railway.
  3. A status message appears: "X record(s) uploaded to Railway successfully".
  4. The Pending Sync counter resets to 0 and records are marked as synced.
  5. Open the Railway portal (cssgrouprms.com) → HR → Attendance — the records will now be there, linked to employees by their Staff ID.
Safe to sync multiple times
Railway uses the (Staff ID + timestamp) pair as a unique key, so uploading the same record twice doesn't create a duplicate — it's safely ignored.
Direct to cloud (skip local)
If you prefer the device to push directly to Railway without local capture, set the device's Server Address to cssgrouprms.com, enable Domain Name (VDomain ON), and leave the Port field hidden (it uses HTTPS 443 automatically). The local app is not needed in this mode.
STEP 12Advanced

Build the .exe (permanent install, no Python needed)

Package the app into a single ZKAttendance.exe that anyone can run without installing Python.

  1. Open Command Prompt as Administrator.
  2. Navigate to the desktop folder: 
    cd C:\RMS\desktop
  3. Run the build script: 
    build.bat
    This installs PyInstaller, generates an icon, and packages everything. Takes 2–3 minutes.
  4. When it finishes, the file is at: 
    C:\RMS\desktop\dist\ZKAttendance.exe
  5. Copy ZKAttendance.exe to wherever you want to keep it (e.g. Desktop, C:\Program Files\).
  6. Double-click it (as Administrator) to run.
Where is the data stored?
When running as an .exe, your attendance database and settings are saved in:
C:\Users\YourName\AppData\Roaming\ZKTecoAttendance\

This folder is never overwritten when you update the app — your data is safe.
STEP 13Advanced

Auto-start with Windows

Make the app start automatically when you turn on the PC, so you never forget to run it on attendance days.

  1. Start the app and look for the tray icon in the bottom-right corner.
  2. Right-click the tray icon.
  3. Click Start with Windows — a tick appears next to it.
  4. From now on, the app starts automatically every time Windows boots.

To turn it off, right-click the tray icon and click Start with Windows again to untick it.

STEP 14Advanced

Update the app

When a new version of the app is released (new features, bug fixes), updating takes 3 commands.

  1. Open Command Prompt in the RMS project folder: 
    cd C:\RMS
  2. Pull the latest changes from GitHub: 
    git pull origin main
  3. Install any new dependencies: 
    cd desktop
pip install -r requirements.txt -q
  4. If you use the .exe, rebuild it: 
    build.bat
    Then replace the old ZKAttendance.exe with the new one from dist\.
Your data is never lost during updates
The attendance database (attendance.db) and settings (settings.json) are stored separately from the app code, so pulling updates and rebuilding never deletes your records.

Troubleshooting

ProblemSolution
App won't start — "Permission denied" or "Port in use" Run Command Prompt as Administrator. Or another app is using port 80 — right-click the tray icon → Settings and change the port to 8080, then update the device's Server Port to match.
Tray icon doesn't appear Look in the hidden icons area (click the ∧ arrow in the bottom-right taskbar). The icon might be hidden. Drag it out to the visible area.
Dashboard doesn't open in browser Wait 5 seconds after starting, then manually open http://localhost or http://127.0.0.1 in your browser.
Diagnostics shows "ZKTeco Device — No connection yet" (1) Confirm the device's Server Address matches your laptop's IP from ipconfig. (2) Confirm Server Port = 80. (3) Confirm Proxy = OFF. (4) Reboot the device and wait 30 seconds.
Scans happen on the device but nothing appears in the dashboard The device's User ID isn't registered in the app. Check the "Unknown Fingerprint" notification — note the Staff ID shown — and register that employee using the Register Employee form.
"Sync failed" when uploading to Railway Check your internet connection. Also confirm the Railway URL in Settings is correct: https://cssgrouprms.com. Run Diagnostics to see if Railway is reachable.
Portal shows employee as Absent even after sync The Staff ID in the app doesn't match the Staff ID in Railway's Employee Directory. They must be exactly the same. Check both and correct whichever is wrong.
Device connects but no heartbeats in diagnostics The device's IP address may be correct but the app wasn't running when it first connected. Reboot the device while the app is running — the first connection request will register it.
build.bat fails with "PyInstaller not found" Run pip install pyinstaller manually, then try build.bat again.
Settings dialog doesn't appear It opens behind other windows. Press Alt+Tab to find it, or look in the taskbar.

Frequently Asked Questions

Can I run this without internet?

Yes. The local server captures attendance to SQLite without any internet. Internet is only needed when you press "Upload to Railway" to sync to the cloud, or during initial package installation.

What happens if the laptop restarts mid-day?

All records that were already captured are safely stored in the SQLite database. Restart the app and the device will reconnect automatically (it retries every 30 seconds). No records are lost — the device also buffers any punches that happened while disconnected and sends them when it reconnects.

Can multiple devices send to the same laptop?

Yes. Set the same Server Address on each ZKTeco device. The app tracks devices by their Serial Number (SN) and processes attendance from all of them together.

Can I run this on multiple laptops simultaneously?

A ZKTeco device can only send to one server address at a time. If you need redundancy, configure the device's Server Address to the Railway cloud URL directly instead.

How do I change the server port from 80?

Right-click the tray icon → Settings → enter the new port (e.g. 8080). Also update the device's Server Port to match. Restart the app for the change to take effect.

Where is the attendance database?

  • Python script mode: C:\RMS\desktop\attendance.db
  • .exe mode: C:\Users\YourName\AppData\Roaming\ZKTecoAttendance\attendance.db

You can open it with any SQLite viewer (e.g. DB Browser for SQLite) to inspect records directly.

How do I back up the attendance data?

Copy the attendance.db file to a safe location (USB drive, cloud storage). The Excel export is also a good daily backup — download one at the end of each day.
